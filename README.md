# json_schema2dart (schema2model)

A `build_runner` builder that turns local JSON Schema documents into strongly typed Dart models. Feed it any `*.json` (or keep the old `*.schema.json`) file and it will emit a sibling `.dart` file with immutable classes, manual `fromJson`/`toJson` methods, local `$ref` resolution, generated enums, and sensible naming.

## Quick start

1. Add this package to your dev dependencies and configure the builder (see `build.yaml`).
2. Drop schemas somewhere under `lib/` or `example/` and give them a `.json` suffix (the legacy `*.schema.json` suffix still works too).
3. Run `dart run build_runner build --delete-conflicting-outputs`.

You'll get Dart source files right next to your schemas. For example, the included `example/schemas/pubspec_subset/schema.json` generates `example/schemas/pubspec_subset/schema.dart` with `Pubspec`, `PubspecEnvironment`, and a `PubspecPublishTo` enum.

Need to narrow which JSON files are processed? Configure `include_globs` on the builder:

```yaml
targets:
  $default:
    builders:
      schema2model|schema_builder:
        options:
          include_globs:
            - example/schemas/**.json
```

By default the builder watches both `**/*.schema.json` and `**/*.json`.

### Builder options

| Option                  | Type / Values        | Default                                       | Description                                                                                       |
|-------------------------|----------------------|-----------------------------------------------|---------------------------------------------------------------------------------------------------|
| `root_class`            | `String`             | inferred from filename / schema `title`       | Force the top-level class name.                                                                   |
| `prefer_camel_case`     | `bool` (`true/false`)| `true`                                        | Whether generated field names favour `camelCase`.                                                 |
| `emit_docs`             | `bool`               | `true`                                        | Include schema `description` values as Dart doc comments.                                         |
| `header`                | `String`             | `"Generated by json_schema2dart"`             | Custom comment header for generated files.                                                        |
| `single_file_output`    | `bool`               | `false`                                       | Emit a single Dart file instead of a multi-part bundle.                                           |
| `allow_network_refs`    | `bool`               | `false`                                       | Permit `$ref` resolution over HTTP(S).                                                            |
| `network_cache_path`    | `String` (path)      | `.dart_tool/schema2model/cache`               | Directory used to cache fetched schemas when network refs are enabled.                           |
| `default_dialect`       | `String` / `"none"`  | latest supported (`https://json-schema.org/draft/2020-12/schema`) | Dialect assumed when a schema omits `$schema`; set to `"none"` to require explicit declarations. |
| `emit_validation_helpers` | `bool`            | `true` (set to `false` to disable)           | Include runtime validation helpers (e.g. `contains`/`minContains` checks).                       |
| `include_globs`         | `String` or `List<String>` | `["**/*.schema.json", "**/*.json"]`     | Limits which input files the builder processes.                                                   |

Because the generator writes manual JSON serializers there is no dependency on a second pass (`json_serializable`, `freezed`, etc.), so the generated models are usable immediately after the build completes.

## Current capabilities

- Detects object, array, string, number, boolean, and integer types.
- Emits immutable classes with value semantics and optional fields marked nullable.
- Reads `required` to decide nullability.
- Resolves local `$ref` targets inside `definitions`/`$defs`.
- Generates `enum`s for string-only `enum` declarations together with helper extensions for `toJson`/`fromJson`.
- Carries schema `description` fields into `///` documentation comments.

See `github-action.json` at the repo root for a real-world schema we plan to support as we add features like discriminator-based unions, `additionalProperties`, and external `$ref` resolution.

## Roadmap

- `oneOf`/`anyOf` → sealed unions (likely via an opt-in `freezed` integration).
- `allOf` composition and flattening.
- Smarter map support for `additionalProperties`/`patternProperties`.
- External `$ref` resolution (relative files and URLs).
- Format hints → rich Dart types (`date-time` → `DateTime`, `uri` → `Uri`, …).
- Optional validation hooks and friendlier doc output (examples, defaults, deprecated flags).

## Development

- Run `dart analyze` to keep the generator clean. Generated `*.schema.dart` files are excluded automatically.
- Exercise the full pipeline with `dart run build_runner build --delete-conflicting-outputs`.

Contributions and feedback are welcome—especially around covering more of the GitHub Actions workflow schema.
